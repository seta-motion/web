What SETA Motion is trying to solve

A practical stop motion workflow should not force you into one camera, one platform, or one fragile pipeline. SETA Motion is built to keep the capture layer flexible while staying close to Blender.

Blender-centered workflow

Work from Blender instead of bouncing between disconnected tools and half-compatible capture apps.

Camera and mobile support

Use dedicated camera drivers where available, while also supporting a mobile backend for lighter setups.

Built for real production needs

Preview, capture, sequence handling, and future stop motion utilities are treated as workflow problems, not shiny gimmicks.

Core features

  • Capture from Blender Operate the shot workflow from inside the animation environment.
  • Live preview support Preview the scene while keeping the capture process aligned with the session.
  • Mobile backend Use an Android device as a networked image/video source for the addon.
  • Driver-based architecture Extend compatibility through device-specific or experimental drivers instead of locking the project to one hardware path.

Near-term roadmap

  • Stronger feature discovery Show only the controls that the active backend or driver actually supports.
  • Driver expansion Broaden the supported command surface without wrecking the stable flows that already work.
  • Preview overlays and onion tools Make previous captures more useful during shooting instead of treating them as a side note.
  • Auxiliary stop motion tools Possibly as a separate addon, so SETA does not become a bloated mess.
